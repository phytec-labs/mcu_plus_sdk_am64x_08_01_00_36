OSAL Migration Guide

This section describes the differences between Driver Porting Layer of MCU+ SDK and OSAL from the Processor SDK RTOS (PDK). This can be used as migration aid when moving from Processor SDK RTOS (PDK) to MCU+ SDK.

Important Notes

  • All DPL APIs use the common return values from APIs for system level define's and function's for success and failure
  • Some APIs take timeout as input, timeout is always specified in units of clock or OS ticks.
    • Use SystemP_NO_WAIT when you dont want to block, i.e timeout of zero.
    • Use SystemP_WAIT_FOREVER when one wants to block for ever, i.e wait forever.
